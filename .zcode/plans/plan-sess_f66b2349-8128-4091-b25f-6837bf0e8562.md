## Gap 1: Rival AI Progression Engine (Task 3 — 2 days)

**Problem:** Rivals are static. `simulateCompetitionDay` generates random RP changes but rivals don't actually *progress* based on player performance or simulated training. The rival activities feed is purely random.

**Solution:**
1. Add `rival-performance-history.ts` — Track rival performance over time (last 10 runs, trends)
2. Enhance `social-store.ts`'s `simulateCompetitionDay` to use progressive difficulty: rivals gain RP faster when the player is ahead, simulating an adaptive AI that responds to player growth
3. Connect result-screen race outcomes to update `rivalRelationships` on the runner profile (wins, losses, relationshipLevel)

**Files to create/modify:**
- `src/social/rival-engine.ts` — NEW: Rival AI progression with training cycle simulation
- `src/social/social-store.ts` — ENHANCE: adaptive difficulty in simulateCompetitionDay
- `src/features/result/result-screen.tsx` — ENHANCE: update rivalRelationships after race
- `src/features/social/social-screen.tsx` — ENHANCE: show rival progression trends

## Gap 2: Comparative Stats & Improvement Tracking (Task 5 — 1 day)

**Problem:** The Stats tab only shows current percentile and static head-to-head. No "vs last run" comparison, no improvement trends.

**Solution:**
1. Create `src/runner/run-history.ts` — Store last N run results (time, grade, distance, date) in a ring buffer per challenge type
2. Add "vs last run" comparison panel to the stats tab showing delta in time, grade, score
3. Add simple trend indicators (↗️ improving, ↘️ declining, ➡️ steady) based on last 5 runs

**Files to create/modify:**
- `src/runner/run-history.ts` — NEW: run history ring buffer
- `src/runner/runner-types.ts` — ENHANCE: add runHistory array to RunnerProfile
- `src/features/result/result-screen.tsx` — ENHANCE: save run to history after race
- `src/features/social/social-screen.tsx` — ENHANCE: add "vs last run" and trend panels to stats tab
