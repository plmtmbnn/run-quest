# MVP Definition

Version: 1.0.0

Status: Draft

Owner: Founder

---

# Purpose

This document defines exactly what belongs in Phase 0.

Everything outside this scope is intentionally postponed.

The objective of Phase 0 is product validation.

Not scalability.

Not monetization.

Not completeness.

---

# Product Hypothesis

We believe that:

Runners enjoy making strategic decisions.

If every day offers one unique race,

players will return daily,

share their stories,

and discuss different strategies.

---

# Success Criteria

Phase 0 is successful if we can answer "Yes" to these questions.

* Do players understand the game in under 60 seconds?
* Do players finish today's race?
* Do players share their results?
* Do players return tomorrow?
* Do players discuss different strategies?

Nothing else matters.

---

# Target Session

A complete session should take:

2–3 minutes.

Maximum:

5 minutes.

# Features Included

## Foundation

* Local Player ID
* Local Storage
* English
* Bahasa Indonesia
* Responsive UI

---

## Daily Race

* One Daily Race
* JSON-based race configuration
* Local race data

---

## Preparation

Players choose:

* Shoes
* Nutrition
* Race Strategy
* Mental Focus

---

## Simulation

* Deterministic
* Checkpoints
* Hidden stats
* Events
* Story generation

---

## Result

Display:

* Finish Time
* Rating
* Story Summary
* Race Analysis

---

## Share

Generate a beautiful Share Card.

---

# Features Excluded

No Login

No Backend

No Database

No Leaderboard

No Friends

No Cloud Save

No Inventory

No Equipment Collection

No Character

No Progression

No Achievements

No Daily Rewards

No Push Notification

No Strava

No GPS

---

# Technical Constraints

Everything must work:

Offline after first load.

Using LocalStorage only.

---

# Definition of Done

Phase 0 is complete when:

A new player can open the website,

finish today's race,

share the result,

and return tomorrow

without requiring an account.

---

# Exit Criteria

Phase 0 ends only after receiving feedback from real players.

No new features should be added before validation.
